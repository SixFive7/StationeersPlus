# Power Grid Plus -- TODO

Backlog for the in-progress mod. Design decisions live in `PLAN.md` (decision log D1-D13); research findings live in the central `Research/` knowledge base. This file is the "consider later / don't forget" list.

## Considered features (not yet decided)

These would all sit on top of the v1 design (the Re-Volt-derived sim half + the three-tier voltage backbone). None of them needs a new prefab, so they stay within the "pure-patch" stance, but each expands the feature surface and needs an explicit go/no-go before implementation.

- [ ] **Cable-heat dissipation (from Deadly Electricity).** Convert a small fraction of a cable network's throughput into heat dumped into the cable run's local atmosphere each tick -- "wires get warm under load". Config-toggled, default off (it's flavour, not a fix). Source of the idea: the [Deadly Electricity](https://steamcommunity.com/sharedfiles/filedetails/?id=3575959825) mod's "cable efficiency loss -> heat with network size" mechanic (see Appendix D). Open questions: how much heat (a small % of `Consumed`?), where exactly to inject it (the cable's `WorldGrid` atmosphere, like `WorldManager.Spark` does on rupture), whether it scales with network size or just throughput, and whether it interacts badly with our (Re-Volt-derived) tick that already does proportional sharing. Decompile the relevant `AtmosphericEventInstance` / `MoleEnergy` injection path before committing.

- [ ] **Absorb Omni Transmitter Settings.** Fold the [Omni Transmitter Settings](https://steamcommunity.com/sharedfiles/filedetails/?id=3643534844) mod's behaviour into Power Grid Plus: configurable max transmit range, distance-based charge-power falloff, and a minimum-power-per-battery floor for the vanilla Omni (Microwave) Transmitter (`PowerTransmitterOmni`). Adds a `Server - Wireless` config section. Notes: this expands Power Grid Plus from "wired power sim" to "wired sim + omni-transmitter tuning" -- a deliberate scope widening (earlier I'd called wireless tuning out of scope; this overrides that for the omni transmitter specifically). It is still a pure patch (patch the existing `PowerTransmitterOmni` class, no new prefab). Check the Omni Transmitter Settings source/license first -- if its source is available and permissively licensed we can adapt it (with attribution); otherwise reimplement the behaviour from scratch (mechanics aren't copyrightable, so a clean reimplementation owes nothing). Don't conflict with this repo's separate Power Transmitter Plus mod, which handles the *Microwave Power Transmitter* dish (a different device from the Omni Transmitter).

- [ ] **Craftable RTG (nuclear recipe).** Add a survival crafting recipe for the RTG (`RadioscopicThermalGenerator`; the prefab already exists and currently only deconstructs as a "Creative RTG" -- confirm the exact prefab name). The recipe should use vanilla nuclear materials (uranium-line ores / refined products -- check what's craftable without a third-party "Processed Uranium" dependency before pulling one in). This is a recipe-only addition (no new prefab), so it fits the pure-patch stance; mechanically it's the same kind of `RecipeData` / gamedata patch as NEW-2's super-heavy-cable cost bump. Precedents for the recipe shape: [CraftableRTG](https://steamcommunity.com/sharedfiles/filedetails/?id=3026880031) (Cu / steel / lead / electrum, fabricator), [Survival RTG](https://steamcommunity.com/sharedfiles/filedetails/?id=2464235697) (lead / steel / inconel / astroloy, Electronics Printer Mk2, 7.5 kJ), [Nuclear Recipes](https://steamcommunity.com/sharedfiles/filedetails/?id=3004097828) (Processed Uranium, expensive/slow). Under D13 a crafted RTG goes on `heavy` cable like every generator -- consistent. Config-toggled. Decide: which machine prints it (Electronics Printer Mk2?), the recipe, and whether it stays "Creative RTG" on deconstruction or returns the kit.

## Verification tasks

- [ ] **Cross-mod compatibility pass (once Power Grid Plus has a working build).** Install Power Grid Plus alongside each power-related mod from the survey (PLAN.md Appendix B / D) and smoke-test, prioritising the compat-flagged cases:
  - [Re-Volt](https://steamcommunity.com/sharedfiles/filedetails/?id=3587239682) -- it swaps the `PowerTick` instance per-network, so our `PowerTick` patches (esp. NEW-1's super-heavy exemption) are bypassed. Decide: detect Re-Volt and refuse to load / patch `RevoltTick` instead / document as incompatible.
  - [MoreCables](https://steamcommunity.com/sharedfiles/filedetails/?id=3555588082) -- adds cable tiers above vanilla. Our three-tier gating must classify them (treat as `superHeavy`-equivalent? reject? configurable?).
  - [CableTypeSwitcher](https://steamcommunity.com/sharedfiles/filedetails/?id=3470776044) and [EL Switche](https://steamcommunity.com/sharedfiles/filedetails/?id=3287183705) -- they create/convert cables at runtime, bypassing our build-time gate; verify the reactive burn-on-join backstop catches the illegal junctions they can produce.
  - [MorePowerMod](https://steamcommunity.com/sharedfiles/filedetails/?id=3243132734) and the big-battery stat mods ([3.6 Megawatt Battery](https://steamcommunity.com/sharedfiles/filedetails/?id=3004087671), [EGC](https://steamcommunity.com/sharedfiles/filedetails/?id=3007388410), [Super XL Wireless Battery (EGC Edit)](https://steamcommunity.com/sharedfiles/filedetails/?id=3706534108), [BuffWirelessBatteries](https://steamcommunity.com/sharedfiles/filedetails/?id=3355912171), [Mod: Jigawatt Battery](https://steamcommunity.com/sharedfiles/filedetails/?id=1785747072), [RRI - Boost Da Powa](https://steamcommunity.com/sharedfiles/filedetails/?id=2359999429)) -- their batteries inherit our discharge-rate limits; check the numbers stay sane (a 230 MW / 1.21 GW battery x 0.7%/tick is a huge wattage; may need a special-case or a config).
  - [Deadly Electricity](https://steamcommunity.com/sharedfiles/filedetails/?id=3575959825) -- patches the same vanilla electrical classes / tick; likely incompatible; confirm and document.
  - [Better Power Mod](https://steamcommunity.com/sharedfiles/filedetails/?id=3234916147), [PowerOverhaul](https://steamcommunity.com/sharedfiles/filedetails/?id=3579306377), [Custom Power DeepMiner](https://steamcommunity.com/sharedfiles/filedetails/?id=3491001740), [Super Structures Mod](https://steamcommunity.com/sharedfiles/filedetails/?id=3159841144), [2x Solar Power](https://steamcommunity.com/sharedfiles/filedetails/?id=1669621266), [Realistic Solar Constants](https://steamcommunity.com/sharedfiles/filedetails/?id=1475742702) -- generation-buff / per-device-tweak mods; expected to coexist, but with generators-on-heavy (D13) a buffed solar/wind output just makes the heavy-tier farm bigger; sanity-check.
  - The rest (recipe/content/economy/QoL mods, IC10 scripts) -- expected fully orthogonal; spot-check a couple.
  - Record results in this repo's `Research/` (a compat page) or the mod's `RESEARCH.md` once it graduates to `Mods/`.
