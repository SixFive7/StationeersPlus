## Allocator follow-ups

- **Battery losses become heat (user-approved 2026-07-08).** Convert the configured charge-efficiency loss (`BatteryChargeEfficiency`) into heat dumped on the battery instead of vanishing, via the existing `EnergyToHeatRatio` hook (Battery inherits it from `ElectricalInputOutput`); today the loss is plain energy destruction, accepted as an interim state when the elastic-to-soft transfer rung shipped. Optionally extend the same mechanism to a configurable fraction of raw (dis)charge throughput (the original shape of this bullet; percentage TBD, default 0 keeps current behaviour). Applies to vanilla Battery, BatteryLarge, and the third-party Nuclear variant via the shared base class.
- **Network-wide battery charge balancing.** Every power tick, batteries on the same cable network redistribute stored charge so all sit at the same `PowerRatio` (an extra pass at the end of ALLOCATE walking each network's battery roster, averaging stored energy proportionally to capacity). Removes the "this one battery is empty while that other is full" failure mode players hit with mixed bank ages. Relation to the elastic-to-soft transfer rung (shipped): the transfer already moves energy between banks whenever a store still has charge headroom (a donor's leftover discharge fills any charging store, cross-network included), so mixed banks now converge while any of them is below full; what it does NOT do is equalize `PowerRatio` between stores that raise no charge request (a full-rate-capped store, or same-net stores whose requests outpace donors), which is the residual gap this bullet would close. Re-evaluate against soak data before building it.
- **Tick-duration watchdog threshold calibration.** The gate-14 four-hour dedicated-server soak (2026-07-08) produced two marginal firings, 152.6 and 149.2 ms against a ~144 ms adaptive threshold, with the allocator's own share at 4-6 ms (ticks 4344 and 11522 of ~29000). Both were environmental (autosave serialization and vanilla spikes) on a box whose tick median sits near 18 ms, so the 8x-median line undercuts the machine's normal spike amplitude. Candidate fixes: raise the 50 ms floor to ~200 ms, raise the multiplier, or warn only when the allocator share is a meaningful fraction of the overrun. Recalibrate against the next soak's data; keep the payload's allocator-vs-total attribution either way.
- **Absorb Omni Transmitter Settings.** Fold the [Omni Transmitter Settings](https://steamcommunity.com/sharedfiles/filedetails/?id=3643534844) mod's behaviour into Power Grid Plus: configurable max transmit range, distance-based charge-power falloff, and a minimum-power-per-battery floor for the vanilla Omni (Microwave) Transmitter (`PowerTransmitterOmni`, a plain single-network consumer per `Research/GameClasses/PowerTransmitterOmni.md`, not a bridge). Adds a `Server - Wireless` config section; still a pure patch. Check the source mod's license first; adapt with attribution or reimplement. Does not overlap Power Transmitter Plus (which handles the dish pair).
