- **Network-wide battery charge balancing.** Every power tick, batteries on the same cable network redistribute stored charge so all sit at the same `PowerRatio` (an extra pass at the end of ALLOCATE walking each network's battery roster, averaging stored energy proportionally to capacity). Removes the "this one battery is empty while that other is full" failure mode players hit with mixed bank ages. Relation to the elastic-to-soft transfer rung (shipped): the transfer already moves energy between banks whenever a store still has charge headroom (a donor's leftover discharge fills any charging store, cross-network included), so mixed banks now converge while any of them is below full; what it does NOT do is equalize `PowerRatio` between stores that raise no charge request (a full-rate-capped store, or same-net stores whose requests outpace donors), which is the residual gap this bullet would close. Re-evaluate against soak data before building it.

- **Shed-downstream-first: the shed flow direction is wrong (user decision 2026-07-12).** The mod's purpose requires that a linear supply chain (solar bank, step-up, transmitter pair, step-down, batteries, step-up, many leaf step-downs) sheds at the LEAVES first: only leaf nets with unmet rigid demand shed, downstream-first, yielding a base with some nets on and some off. The trunk (solar up through the step-ups) must never fault, since that chain can exist perfectly with no practical load. Today victim selection is per-net among that net's claimants, so a sole sheddable claimant on a mid-chain hop net (the transmitter pair, or the lone step-down) can shed WHOLE when a sustained deficit surfaces at its hop, blacking out everything below it, and whether the backward desire pass nets out mid-chain battery elastic before the deficit reaches an upstream hop is UNVERIFIED (build a ScenarioRunner chain fixture: synthetic seven-link chain, throttle the solar, log which seg sheds at which round). Fix direction: deficit forwarding, every non-leaf hop behaves like a step-up (partial grant, never a victim) so the deficit always lands where multiple fine-grained victims exist; formalize "sheddable = leaf step-down with unmet rigid demand". Taken into account in the 2026-07-12 architecture redesign (see `.work/2026-07-12-architecture-redesign/SESSION-HANDOFF.md`).
