# Changelog

Full version history for Power Grid Plus. The newest entry also appears in `About.xml` `<ChangeLog>`.

## v0.2.0: Atomic power tick and the fault system (work in progress)
- The power simulation is fully mod-owned: one per-tick snapshot reads the whole grid exactly once, the global allocator solves every network at once, and a write-back applies the result (delivered energy, stored charge, Powered flags, network readouts, fuse protection, cable burns). The vanilla per-network power flow no longer runs, removing its mid-tick races, billing leaks, and partial-power brownouts by construction: a machine is either fully powered or its network is cleanly dark with a visible fault.
- Load shedding is leaf-first: only bridges feeding end-of-chain networks with unmet demand shed, lowest Priority first; mid-chain hops forward the deficit downstream instead of decapitating a whole supply chain, and an unloaded trunk chain never sheds.
- Four 60-second fault lockouts with on-device feedback: Shedding (orange flash; the input network cannot cover the device, lowest Priority sheds first), Overloaded (red; delivering at the cap with demand still unmet), Cycle Fault (red; every loop member faults and the loop dissolves without a burn), and Variable Voltage Fault (red; an unprotected producer stops generating). Hover text names the cause with a live countdown; toggling the device off clears the fault instantly.
- Producer isolation is always on and strict: producers (solar, wind, generators, RTG, turbine, dynamic generator dock) may only share a cable network with other producers and transformers. Anything else on the network faults the producers, batteries and other bridging devices included, and a transformer on the network does not exempt the rest. Wire generator banks through a transformer (the Medium transformer takes heavy cable on both sides). Unrecognised modded producers fall back to burning the cable next to the violating device.
- Elastic storage: batteries, APC cells, and rocket power umbilicals charge only from surplus and discharge only to fill a shortfall, with proportional sharing among competitors. New read-only logic values `MaxChargeSpeed`, `MaxDischargeSpeed`, `ChargeSpeed`, and `DischargeSpeed` on all storage devices; the earlier Import Quantity / Export Quantity exposure on batteries is removed (breaking change for scripts that read them).
- Transformer Setting is vanilla again: IC10 reads and writes the live throughput cap (the earlier redirect of Setting writes to Priority is removed); new transformers start at full throughput, saved worlds keep their saved Settings, and a Setting of 0 means deliberately off (logged at load). The knob and the Labeller set Priority (no upper cap). Dispatch is per input network by Priority.
- Cable Watts caps are configurable per tier (normal 5000, heavy 100000 matching vanilla, super-heavy unlimited; 0 = unlimited) and enforced at runtime only; cables in the save are never modified, so removing the mod restores vanilla ratings.
- Only direct generator overflow burns cables; transformer or battery overflow trips the upstream devices into Overloaded instead. Burned-cable reasons now survive save and load (side-car).
- Power transmitter / receiver pairs participate in dispatch as transformer-like devices, using whichever distance-loss model is active (vanilla curve or PowerTransmitterPlus).
- Rocket power umbilical rate caps and logic values (master toggle plus charge / discharge rate settings).
- Multiplayer fault sync rebuilt as per-tick full snapshots with a join handshake, so clients always show the correct flash and countdown, self-healing after packet loss.
- Emergency-light target list: the battery wall-light emergency behaviour now applies to a configurable prefab list (default StructureWallLightBattery).
- Removed settings: Enable Voltage Tiers, Enable Recursive Network Limits, Enable Unlimited Super-Heavy Cables, Enable Transformer Exploit Mitigation, and Enable APC Power Fix (all now always on; the super-heavy toggle became the 0 = unlimited cap; orphaned config entries are ignored). New settings: the three per-tier cable caps, APC Battery Discharge Rate, the three Rocket Umbilical entries, Enable Transformer Overload Protection, and Emergency Light Prefabs.
- Devices no longer power-cycle on their own demand spikes: the mod decides every device's Powered flag from its network's state (live, shed, overloaded, or dead), so a printer starting a job cannot reboot itself and cancel the print. Either the subnet carries the load, or it goes dark as a unit for 60 seconds and delivers nothing while dark.
- Powered and the on/off switch are now independent (new setting Decouple Powered From On Off, default on): a switched-off device on a live network reads Power=1 in IC10 and draws nothing. Read On for the switch state; turn the setting off to restore the vanilla coupling. The ownership itself has no toggle.
- Every device's demand and output is read exactly once per tick into the grid snapshot, so a mid-tick change can never tear the solve; the fabricator-family work accumulators are drained on the thread that owns them, and unfunded work on a dark network freezes and is billed in full on revival.

## v0.1.0: First build (work in progress)
- Reworked cable-network power tick from Re-Volt: proportional load sharing, gradual probabilistic cable burnout on a rolling throughput average, NaN-power guard.
- Stationary battery charge/discharge-rate limits, optional charge efficiency, and rate-limit logic values.
- Transformer free-power exploit fix, quiescent-draw restore, and Power Actual logic value.
- Area Power Controller power-leak / idle-drain fix.
- Recursive and looped networks allowed by default.
- Super-heavy cable never burns; the three cable tiers are separate transmission voltages (mixing them burns the lower-tier boundary cable and splits the network); devices belong on their tier (generators and batteries on heavy, high-draw machines on heavy or normal, super-heavy is cables-and-transformers only, everything else on normal) and a device on the wrong tier gets no power; super-heavy cable coil costs 2x to craft (configurable via the multiplier).
- Reactive tier enforcement: a wrong-tier cable burns on the next power tick if the network has actual power flow (an idle or off network destroys nothing). Wrong-tier cable placement is blocked at the cursor with a "Wrong voltage" tooltip; device placement is never blocked. Burned cables explain why they burned in their hover tooltip.
- Logic passthrough: a writable `LogicPassthroughMode` logic value makes power-network bridging devices logic-transparent, so a logic reader sees devices on the far side, transitively across a whole chain of bridges (cycle-safe). Covers transformers, stationary batteries, linked power transmitter and receiver dishes, and Area Power Controllers. Per-device mode is server-authoritative, persists across save and load, and each device family has its own master server toggle.
- Transformer priority and load shedding: transformer throughput is hardcoded at the per-prefab `OutputMaximum` rating; the knob now sets a writable `Priority` value (default 100, step 10 per click or 1 with Alt). Allocation on a shared input network is strictly by `Priority`; a transformer that cannot get its full share sheds for 10 seconds (contributes 0, orange pulse, hover explains), then auto-clears. New writable `Priority` and read-only `Shedding` logic values; `Setting` writes redirect to `Priority` so legacy IC10 scripts keep working. Priority saves with the world and syncs in multiplayer. Master toggle `Enable Transformer Shedding` (default on).
- Deterministic network merge: when cable or structure networks merge, the survivor is chosen by lowest reference id so the host and all clients agree on the same network id (a multiplayer correctness fix).
