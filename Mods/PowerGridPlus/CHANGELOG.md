# Changelog

Full version history for Power Grid Plus. The newest entry also appears in `About.xml` `<ChangeLog>`.

## v0.2.0: Atomic power tick and the fault system (work in progress)
- The power simulation is fully mod-owned: one per-tick snapshot reads the whole grid exactly once, the global allocator solves every network at once, and a write-back applies the result (energy, storage, Powered flags, readouts, fuses, burns). The vanilla per-network power flow no longer runs, so its mid-tick races, billing leaks, and partial-power brownouts are gone.
- Deprioritization is leaf-first: only bridges feeding end-of-chain networks with unmet demand are deprioritized, lowest Priority first with the smallest-combined-watts cut inside a tier; mid-chain hops forward the deficit downstream, and step-up transformers compete like any branch (give backbone transformers a high Priority to shed them last).
- Five 60-second fault lockouts: Deprioritized (orange flash; the input runs short, the lowest Priority drops first), Device overloaded (red; more draw than the device can deliver, titled per device: Transformer / Link / Battery / APC / Umbilical), Cable overloaded (red; the weakest cable cannot carry the flow), Cycle fault (red; every loop member faults, no burn), and Current mismatch (red; an unprotected producer stops generating). Toggling off clears instantly.
- One hover template on the casing and the on/off button: the switch state and the fault title with a live countdown, then grey diagnostics with the offending value in red and the capacity in blue (Drawing 150.0 kW of 100.0 kW, Pushing 80.0 kW onto a 5.0 kW wire, Needs 12.0 kW while 30.0 kW competes for 20.0 kW upstream). The current-mismatch hover names the offenders, one per line, capped at three.
- Five read-only fault logic values: `DeprioritizedFault`, `DeviceOverloadedFault`, `CableOverloadedFault`, `CycleFault`, and `CurrentMismatchFault`, each 1 while its lockout is active. A battery tripped by cable overflow now takes the visible 60-second lockout instead of silently producing zero.
- Producer isolation is always on and strict: producers may only share a cable network with other producers and transformers; anything else faults them, and a transformer exempts nothing. Unrecognised modded producers fall back to burning the adjacent cable.
- Elastic storage: batteries, APC cells, and rocket power umbilicals charge only from surplus and discharge only to fill a shortfall, sharing proportionally. New read-only logic values `MaxChargeSpeed`, `MaxDischargeSpeed`, `ChargeSpeed`, and `DischargeSpeed` on all storage devices; the Import Quantity / Export Quantity exposure on batteries is removed (breaking change).
- Battery Charge Efficiency is now a charge cost: grid energy drawn per unit stored, default 1.5 (stores two thirds of the draw; values below 1.0 count as 1.0). Post-loss trickles below 500 W store in full.
- Transformer Setting is vanilla again: IC10 reads and writes the live throughput cap; new transformers start at full throughput, saved worlds keep theirs, and Setting 0 means deliberately off. The knob and the Labeller set Priority; dispatch is per input network by Priority.
- Cable Watts caps are configurable per tier (normal 5000, heavy 100000, super-heavy 0 = unlimited) and enforced at runtime only; cables in the save are never modified.
- Only direct generator overflow burns cables; transformer or battery overflow trips the upstream devices into the cable-overloaded fault instead. Burned-cable reasons survive save and load.
- Power transmitter / receiver pairs participate in dispatch as transformer-like devices, using whichever distance-loss model is active (vanilla or PowerTransmitterPlus).
- Rocket power umbilical rate caps and logic values (always on).
- Multiplayer fault sync rebuilt as per-tick full snapshots with a join handshake, so clients always show the correct flash, countdown, and hover numbers.
- Emergency lights: the battery wall-light emergency behaviour applies to a configurable prefab list.
- Logic passthrough support is always on; a device bridges only while its LogicPassthroughMode is 1. Six new Passthrough Default settings seed a device whose mode was never set; a stored mode wins once set. Defaults: transparent except non-small transformers; the APC (no logic port) is config-seeded only.
- Removed settings: every on/off master for behaviour that is now always on, plus the Powered decoupling toggle; the super-heavy toggle became the 0 = unlimited cap; orphaned entries are ignored. New settings: the per-tier cable caps, APC Battery Discharge Rate, the Rocket Umbilical rates, the six Passthrough Default seeds, Extra Delivery Devices, and Emergency Light Prefabs.
- Devices no longer power-cycle on their own demand spikes: the mod decides every device's Powered flag from its network's state, so a printer starting a job cannot reboot itself and cancel the print.
- Powered still follows the on/off switch: a switched-off device goes dark exactly as in the base game; a toggle lands within one power tick. In IC10, Power on a plain device now reads network liveness (1 on an energized network, even while switched off); read On for the switch state.
- Devices that do their work when power arrives keep working: the Omni Power Transmitter, Suit Storage, Battery Cell Charger, Powered Bench, and Wall Light Battery receive their granted power (the wall light also drives the emergency-light logic). Extra Delivery Devices enrolls modded devices; the load log names candidates.

## v0.1.0: First build (work in progress)
- Reworked cable-network power tick from Re-Volt: proportional load sharing, gradual probabilistic cable burnout on a rolling throughput average, NaN-power guard.
- Stationary battery charge/discharge-rate limits, optional charge efficiency, and rate-limit logic values.
- Transformer free-power exploit fix, quiescent-draw restore, and Power Actual logic value.
- Area Power Controller power-leak / idle-drain fix.
- Recursive and looped networks allowed by default.
- Super-heavy cable never burns; the three cable tiers are separate transmission voltages (mixing them burns the lower-tier boundary cable and splits the network); devices belong on their tier (generators and batteries on heavy, high-draw machines on heavy or normal, super-heavy is cables-and-transformers only, everything else on normal) and a device on the wrong tier gets no power; super-heavy cable coil costs 2x to craft (configurable via the multiplier).
- Reactive tier enforcement: a wrong-tier cable burns on the next power tick if the network has actual power flow (an idle or off network destroys nothing). Wrong-tier cable placement is blocked at the cursor with a "Wrong voltage" tooltip; device placement is never blocked. Burned cables explain why they burned in their hover tooltip.
- Logic passthrough: a writable `LogicPassthroughMode` logic value makes power-network bridging devices logic-transparent, so a logic reader sees devices on the far side, transitively across a whole chain of bridges (cycle-safe). Covers transformers, stationary batteries, linked power transmitter and receiver dishes, and Area Power Controllers. Per-device mode is server-authoritative, persists across save and load, and each device family has its own master server toggle.
- Transformer priority and deprioritization: transformer throughput is hardcoded at the per-prefab `OutputMaximum` rating; the knob now sets a writable `Priority` value (default 100, step 10 per click or 1 with Alt). Allocation on a shared input network is strictly by `Priority`; a transformer that cannot get its full share is deprioritized for 10 seconds (contributes 0, orange pulse, hover explains), then auto-clears. New writable `Priority` and read-only `DeprioritizedFault` logic values; `Setting` writes redirect to `Priority` so legacy IC10 scripts keep working. Priority saves with the world and syncs in multiplayer. Master toggle (default on; removed in v0.2.0, always on).
- Deterministic network merge: when cable or structure networks merge, the survivor is chosen by lowest reference id so the host and all clients agree on the same network id (a multiplayer correctness fix).
