## Playtest checklist (before publishing v1.1.0)

Verified on the dedicated server (so NOT on this list): the on-load long-piece rebuild (40 long pieces -> 198 single tiles, 0 failures), the on-load straight-cable roll normalisation (1037 cables realigned, all run axes end at the single canonical rotation, persisted in `RegisteredWorldRotation`), networks not orphaned, and the server starting cleanly with `MOD.Networking.Required = true` + the `IJoinValidator`. The items below need an interactive client (and a second one for the multiplayer item) and can't be reasoned to a certainty:

- [ ] **Merge-with-tool does not throw.** Hold a gas-pipe kit + its exit tool, aim at an existing straight pipe run, merge -- no `KeyNotFoundException` / no error (this is the latent v1.0 regression the `AutomaticSetup` pass fixes). Repeat with a cable kit. Also: normal single-tile pipe/cable placement still works, and the C-key smart-rotate on a pipe/cable cursor cycles 3 orientations with no per-frame exception in `Player.log`.
- [ ] **Build a long piece, see it rewritten.** Spawn or build a `StructurePipeStraight10` (or `StructureCableSuperHeavyStraight10`) via the creative menu or NetworkUpgrader's `upgrade` -- it should be replaced immediately by N single tiles (a `[NetworkPuristPlus]` "built N x ..." console line), networks intact, no error. (Gas in the rewritten pipe is gone -- expected, by design.)
- [ ] **Build straight cables, check the roll.** Place a short straight-cable run by plain clicking; press the rotate key between two of the placements; also zoop a stretch (ZoopMod); also paste a small blueprint of a cable run (BlueprintMod). - [ ] **Multiplayer join enforcement.** With the host running NetworkPuristPlus v1.1.0, try to connect a second client (a) without the mod, (b) with a different setting flipped -- both should be rejected at join with a clear `[NetworkPuristPlus]` message; a client with the mod + the same settings joins fine and sees the host's rebuilt/realigned world.

## Won't do (decided against)

- **Carry chute items in transit over to the rebuilt single-tile chutes.** An item physically inside a long chute segment at the moment it is rebuilt is deleted, by design. The README states this.
- **Preserve a rebuilt pipe run's gas.** The gas inside a long pipe run is deleted when the run is rebuilt from single tiles, by design (the game's atmospherics-event system loses a pipe network's gas when its pipes are replaced this way; capture-then-`Atmosphere.Add` was tried and did not stick). The README states this. Re-pressurise rebuilt pipe runs.
