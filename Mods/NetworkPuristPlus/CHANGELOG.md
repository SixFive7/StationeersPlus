# Changelog

Full version history for Network Purist Plus. The newest entry also appears in `About.xml` `<ChangeLog>` and as the latest note on the Steam Workshop Change Notes tab.

## v1.1.3: Guard against the ZoopMod long-piece merge crash
- Added a safety guard for a crash that can hit when ZoopMod drag-builds a pipe or cable across an existing long straight piece (the 3, 5, and 10 segment variants). The game computes a negative item cost for that merge and the build crashes; the guard clamps the cost to zero so the build continues. The root cause is in ZoopMod and the base game and has been reported to the ZoopMod author. The guard runs whenever the mod is enabled, including with the long-piece removal toggles turned off, which is the setup that leaves long pieces in the game to trip over.

## v1.1.2: Settings descriptions corrected
- Reworded the in-game settings descriptions, and the matching note in the Workshop description, so they no longer mention the build-time long-piece rewrite that was removed in v1.1.1. The behaviour is unchanged: long pieces are taken out of the build kits and rebuilt from single tiles when a save loads.

## v1.1.1: Multiplayer cable-alignment fix
- A freshly built straight cable is set to the canonical orientation before it is created, instead of just after it registers. In multiplayer the cable now shows the same orientation on every player's screen the instant it is built; the previous version aligned it just after creation, which could leave a just-built cable mis-rolled on other players' screens until the next world load. Covers normal placement, the rotate key, ZoopMod drag-builds, and the merge-with-tool action. A blueprint paste (BlueprintMod) is realigned on the next world load instead, by the on-load sweep.
- Cable alignment runs on the host (or in single-player) only, as a safeguard so a client never changes a cable's rotation locally.
- Clarified the description: a long piece that enters a world some other way (a blueprint paste, NetworkUpgrader, the creative menu, another mod) is converted to single tiles on the next world load. The long variants are removed from the build menus, so there is no way to build one directly.

## v1.1.0: Settings panel, network enforcement, cable alignment, build-time long-piece rewrite
- Added a settings panel (in the in-game mod settings, under Server - Pieces and Server - Cables): a master Enable Network Purist Plus toggle, per-family long-piece toggles (Remove Long Gas Pipes, Remove Long Liquid Pipes, Remove Long Insulated Gas Pipes, Remove Long Insulated Liquid Pipes, Remove Long Chutes, Remove Long Super-Heavy Cables), and an Align Straight Cables toggle. All default on. All server-authoritative, so the host's values take effect. All of them take effect on a game restart: the strip / hide / build-cursor work runs once at startup, so toggling one mid-session does nothing until you relaunch (the in-game panel marks them with a restart indicator). The pipe-family toggle names spell out "Gas Pipe" / "Insulated Gas Pipe" even though the game's own term for the basic pipe is just "Pipe", otherwise you couldn't tell the gas toggle from the liquid one.
- Network enforcement (via LaunchPadBooster): every player and dedicated server on a multiplayer game must run the same version AND the same settings. A client without the mod, with a different version, or with different settings is rejected at join time with a clear message. Necessary because the build-kit options and the Stationpedia are reshaped at prefab-load time, before any client joins.
- Straight cables (all tiers) are aligned to one consistent orientation per run axis: every already-placed straight cable is realigned when a world loads, and a freshly built cable snaps to the canonical orientation the moment it is placed. Fixes the misaligned-band look on cable runs (most visible on super-heavy cables, common where zoop-built and hand-built cables meet). Cosmetic only: connectivity, networks and colour are untouched. The rotate key on a cable becomes preview-only. The chosen orientation is the one that mates with the most corner pieces; the coloured band can still jump where a straight meets a corner cable (corner pieces have a fixed band orientation and aren't re-rolled, and no single per-axis orientation mates with every corner), so a full corner-seam fix is planned as a future change.
- A long piece that appears mid-game (a blueprint paste, another mod) is now expanded into single tiles the moment it is built, instead of only being caught on the next world load.
- The single-tile straight pipe, cable and chute in each build kit is given the "straight" connection type, which makes the build cursor and the smart-rotate cycle consistent and keeps the merge-with-tool action working now that the long straight variants are gone from the kits (without that, merging two collinear pipes or cables with the tool throws an error).
- Belt-and-suspenders: a Harmony postfix on the Stationpedia page-override pass re-asserts the hidden flag on the removed long variants, in case a future mod's page override would clear it.
- Compatible with ZoopMod, BlueprintMod, NetworkUpgrader and other build mods: no patch conflict (this mod's Harmony patches are on methods none of them patch), and the mod never rejects anything; it rewrites a long-variant placement into single tiles (at build time, or on the next world load) and re-rolls a built straight cable, so those mods keep working and just produce single-tile, consistently oriented pieces.

## v1.0.0: Initial release
- Removes the long straight pipe, insulated pipe, liquid pipe, insulated liquid pipe, chute, and super-heavy cable variants (the 3, 5, and 10-segment pieces) from the build-kit mouse wheel and hides them from the Stationpedia.
- On save load, rebuilds every already-placed long run from single-tile pieces at the same cells, rotation, and colour, keeping networks connected. Rebuilt pipe runs start empty (gas contents are not preserved); re-pressurise them.
- Compatible with ZoopMod (zooping a long stretch produces single-tile pieces) and NetworkUpgrader.
- Known limitations: a rebuilt pipe run starts empty; an item physically in transit inside a long chute segment is lost when that segment is rebuilt.
